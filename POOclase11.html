La composición es un caso especial de la relación
de agregación. 
El modelo de agregación tiene una relación que 
representa
una relación de propiedad de dos objetos. El objeto
 propietario
se llama objeto de agregación y su clase
 se llama clase de
agregación. El objeto sometido se llama objeto agregado y su
clase se llama clase agregada. Un objeto
 agregado, pude ser
objeto agregado de varias clases de agregación.
Si un objeto es exclusivamente agregado a un objeto de 
agregación, 
la relación entre los objetos es una relación de 
composición.




La clase IndiceDeMasa contiene un campo de datos String. La 
relación entre la clase IndiceDeMasa y el campo de datos String
es de composición.
 


Composición y agregación



La agregación de composición, es cuando una clase es el 
todo y la otra clase es parte del todo. Con la restricción 
de que la clase parte es exclusiva de la clase todo. En un 
diagrama de clases el diamante lleno representa una relación
de composición.

El programa "Fecha.java"




public class Fecha {
   private int dia;
   private int mes;
   private int anio;

   public Fecha() {
      dia = 1;
      mes = 1;
      anio = 1998;
   }

   public Fecha(int d1, int m1, int a1) {
      dia = d1;
      mes = m1;
      anio = a1;
   }

   public boolean equals(Object ob1) {
      if(!(ob1 instanceof Fecha))
         return false;
      Fecha ob2 = (Fecha) ob1;
      return ob2.dia == dia && ob2.mes == mes && ob2.anio == anio;
   }

   public String toString() {
      return dia + "/" + mes + "/" + anio;
   }
   
   public static void main(String[] args) {
      Fecha f1 = new Fecha(28,2,2022);

      System.out.println(f1.toString());
     
   
   }
}
