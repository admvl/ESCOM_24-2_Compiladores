USO DE VARIABLES Y MÉTODOS DE CLASE

Objetivo

Se creara una clase, donde se aplicará el uso de las variables y métodos de clase.

Actividades


Variables de clase. Cuando piensa en una solución orientada a objetos, ¿qué visualiza? Basado en lo que ha aprendido hasta este punto, debería ver objetos separados, cada uno con su propio conjunto de datos y comportamientos (variable de instancia y métodos de instancia, respectivamente). Esa es una imagen válida, pero debe tener en cuenta que, además de datos y comportamientos que son específicos de objetos individuales, también puede tener datos y comportamientos relacionados con toda una clase. Dado que se relacionan con una clase completa, dichos datos y comportamientos se denominan variables de clase y métodos de clase, respectivamente.


Las clases permiten que un diseñador de software considere los objetos como diferentes tipos de entidades. Visita objetos de esta manera nos permite utilizar los mecanismos de clasificación para categorizar estos tipos, definir jerarquías y comprometerse con las ideas de especialización y generalización de objetos.


Para convertir una variable en una variable de clase, use el modificador static en su declaración. El modificador static es por qué muchos programadores usan el término "variable estática" cuando hablan de variables de clase. Asimismo, desde las constantes de clase y los métodos de clase también usan el modificador static, muchos programadores usan los términos static método constante y estático. Utilizaremos los términos variable de clase, constante de clase y método de clase, ya que esos son los términos que utiliza Sun. Aquí está la sintaxis para una declaración de declaración de variable de clase:


Forma general de la declaración de variables de clase.


       private-o-public static tipo nombre-variable;

Como sabe, cada vez que se usa new se crea una copia separada de todas las variables de instancia para cada objeto. Las variables de clase son diferentes. Para una clase en particular, solo hay una copia de cada variable de clase y todos los objetos comparten esa única copia. Por lo tanto, debe usar variables de clase para describir las propiedades de los objetos de una clase que necesitan ser compartido por todos los objetos. Por ejemplo, considere nuevamente el problema de simular el crecimiento del ratón. En nuestro programas de Raton.java anteriores, realizamos un seguimiento de los datos pertinentes a cada ratón individual: la tasa de crecimiento de un ratón, la edad de un ratón y el peso de un ratón. Métodos de clase: Los métodos de clase, como las variables de clase, se relacionan con la clase como un todo y no se relacionan con objetos individuales. Como tal, si necesita realizar una tarea que involucre a la clase como un todo, entonces debe implementar y utilizar un método de clase. Los métodos de clase definidos en la clase Java API Math; por ejemplo, Math.round y Math.sqrt. Los métodos de clase a menudo acceden a variables de clase, y al escribir sus propios métodos de clase, tendrá la oportunidad de ver cómo acceder a las variables de clase que ha definido.


Si desea que todas las instancias de una clase compartan datos, use variables estáticas, también conocidas como variables de clase. Las variables estáticas almacenan valores para las variables en una ubicación de memoria común. Debido a esta ubicación común, si un objeto cambia el valor de una variable estática, todos los objetos de la misma clase se ven afectados. Java admite métodos estáticos y variables estáticas. Los métodos estáticos (métodos de clase) se pueden llamar sin crear una instancia de la clase.


Ejemplo 2. El método de clase es numeroDeObjeto:


       class Circulo2 {
         /** El radio del circulo */
         double radio;
         /** El número de objeto creado */
         static int numeroDeObjeto = 0;
           /** Constructor de un circulo con radio 1 */
         Circulo2() {
           radio = 1.0;
           numeroDeObjeto++;
         }
         /** Constructor de un circulo con un radio especifico */
         Circulo2(double radioNuevo) {
           radio = radioNuevo;
           numeroDeObjeto++;
         }
         /** Retorno de numeroDeObjeto */
         static int getNumeroDeObjeto() {
           return numeroDeObjeto;
         }
         /** Retorno del área del ciculo */
         double getArea() {
         return radio * radio * Math.PI;
         }
       }

       public class TestCirculo2 {
         /** Metodo main */
         public static void main(String[] args) {
           System.out.println("Antes de crear el objecto");
           System.out.println("El numero del objeto Circulo es " +
             Circulo2.numeroDeObjeto);

           // Crear c1
           Circulo2 c1 = new Circulo2();

           // Desplegar c1 ANTES DE crear c2
           System.out.println("\nDespues de crear c1");
           System.out.println("c1: radio (" + c1.radio +
               ") y el numero de objeto circulo (" +
                c1.numeroDeObjeto + ")");
           // Crear c2
           Circulo2 c2 = new Circulo2(5);
           // Modificar c1
           c1.radio = 9;
           // Desplegar c1 y c2 DESPUES c2 fue creado
           System.out.println("\nDespues de crear c2 y modificar c1");
           System.out.println("c1: radio (" + c1.radio +
            ") y numero de objeto Circulo (" +
           c1.numeroDeObjeto + ")");
           System.out.println("c2: radio (" + c2.radio +
             ") y numero de objeto Circulo (" +
           c2.numeroDeObjeto + ")");
         }
       }
