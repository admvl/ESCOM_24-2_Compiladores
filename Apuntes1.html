Sobre multiclases, clases abstractas e interfaces




Una interfaz se trata como una clase
especial en Java. Cada interfaz se
compila en un archivo bytecode por
separado, como una clase normal. Al 
igual que con una clase 
abstracta, no se puede crear un
instancia desde una interfaz con el uso 
del operador new, pero en la mayoría 
de los casos puede usar una interfaz
más o menos de la misma manera que usa 
una clase abstracta. Por ejemplo, 
puede usar una interfaz como
un tipo de datos para una 
variable de referencia, como resultado 
de la conversión, etc.
Ahora puede usar la interfaz Edible
para especificar si un objeto es 
comestible. Esto es
logrado al permitir que la clase 
para el objeto implemente esta 
interfaz usando la palabra 
clave implements. Por ejemplo, las 
clases Chicken y Fruit en el Listado 
14.6 (líneas 14, 23) implementan la 
interfaz Edible. La relación entre la 
clase y la interfaz se conoce como 
herencia de interfaz. Dado que la 
herencia de interfaz y la herencia de 
clase son esencialmente las mismas, 
simplemente nos referiremos a ambas 
como herencia.

Programa sobre la interface
relación. Libro: Datan [pág. 212]



Con la palabra clave interface, puede 
abstraer completamente la interfaz de 
una clase desde su implementación.
Es decir, utilizando interface, 
puede especificar qué debe hacer una 
clase, pero no cómo lo hace. Interfaces 
son sintácticamente similares a las 
clases, pero carecen de variables de 
instancia y sus métodos son
declarados sin ningún cuerpo. En la 
práctica, esto significa que puede 
definir interfaces que no hacen
suposiciones sobre cómo se 
implementan. Una vez que se define, 
cualquier número de Las clases pueden 
implementar una interface. 
Además, una  clase puede implementar 
cualquier cantidad de interfaces.
Para implementar una interfaz, una 
clase debe crear el conjunto completo 
de métodos definidos por
la interfaz. Sin embargo, cada clase 
es libre de determinar los detalles 
de su propia implementación.
Al proporcionar la palabra clave 
interface, Java permite utilizar 
completamente la "interfaz única,
métodos múltiples" aspectos del 
polimorfismo.



Una vez que se ha definido una interface, 
una o más clases pueden implementar esa 
interfaz. Para implementar una interfaz, 
incluir la cláusula implements en 
una definición de clase y luego crear
los métodos definidos por la interfaz. 
La forma general de una clase que incluye 
implements. La cláusula se ve 
así [manual java, pág. 194]:



Puede declarar variables como 
referencias de objeto que usan 
una interfaz en lugar de un tipo 
de clase. Cualquier instancia de 
cualquier clase que implemente la 
interfaz declarada puede ser 
referida por tal una variable. 
Cuando llame a un método a través 
de una de estas referencias, la 
versión correcta se llamará en 
función de la instancia real de 
la interfaz a la que se hace 
referencia. éste es uno de
características clave de las 
interfaces. El método a ejecutar 
se busca dinámicamente en tiempo 
de ejecución,
permitiendo que las clases se creen 
más tarde que el código que llama 
métodos en ellas. La llamada
el código puede enviarse a través 
de una interfaz sin tener que saber 
nada sobre el "destinatario".
Este proceso es similar al uso de 
una referencia de superclase para 
acceder a un objeto de subclase, 
como se describe en el Capítulo 8.

Las clases abstractas son como clases regulares
con datos y métodos, pero no puede crear
instancias de clases abstractas utilizando el
operador new.




Un método abstracto no puede estar contenido en
una clase no abstracta. Si una subclase de un
superclase abstracta no implementa todos los
métodos abstractos heredados de la superclase,
la subclase debe definirse abstract.




Una clase que contiene métodos abstractos debe
ser abstract. Sin embargo, es posible definir
una clase abstracta que no contiene métodos
abstractos.



Una subclase puede ser abstracta incluso si su
superclase es concreta.

Una interfaz es una construcción de clase que
contiene solo constantes y métodos abstractos.
En muchos sentidos, una interfaz es similar a
una clase abstracta, pero una clase abstracta
puede contener constantes y métodos abstractos,
así como variables y métodos concretos.
