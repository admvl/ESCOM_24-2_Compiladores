Los tres pilares
de la programación orientada a ojetos. 
Libro: Liasing [pág. 384]


Los tres pilares de la programación orientada 
a objetos son encapsulación, herencia y
polimorfismo. Ya has aprendido los dos 
primeros. Esta sección presenta polimorfismo.
Primero definamos dos términos útiles: 
subtipo y supertipo. Una clase define un 
tipo. Un tipo definido por una subclase se 
llama subtipo y un tipo definido por su 
superclase se llama supertipo. Entonces, 
puedes decir que Circle es un subtipo de 
GeometricObject y GeometricObject es un
supertipo para Circle.






Sobreescribiendo
métodos. Libro: Liasing [pág. 382]


Una subclase hereda métodos de una
superclase. Algunas veces es necesario
en las subclases modificar la
implementación de un método
definido en la superclase. Esto
es un método sobreescrito.

El método toString en la clase 
GeometricObject devuelve la 
representación de cadena
para un objeto geométrico. 
Este método se puede anular para 
devolver la representación de 
cadena para un circulo. Para 
anularlo, se agrega el siguiente 
método nuevo en Circle4.java:

public class Circle4 extends GeometricObject1 {
  // Otros metodos son omitidos

  /** Sobreescribir el metodo toString definido en
      GeometricObject */
  public String toString() {
    return super.toString() + "\nel radio es " + 
     radius;
  }
}
