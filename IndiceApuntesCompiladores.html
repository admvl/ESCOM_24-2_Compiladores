Enlances a apuntes

La programación orientada a objetos le permite derivar nuevas clases de clases existentes. Esto es llamado herencia. La herencia es una característica importante y poderosa en Java para reutilizar software. Suponga que debe definir clases para modelar círculos, rectángulos y triángulos. Estas clases tienen muchas características comunes. ¿Cuál es la mejor manera de diseñar estas clases para evitar redundancia y hacer que el sistema sea fácil de comprender y fácil de mantener? La respuesta es a usar la herencia.

Se utiliza una clase para modelar objetos del mismo tipo. Las diferentes clases pueden tener algunas propiedades y comportamientos, que pueden generalizarse en una clase que pueden compartir otros clases, la herencia le permite definir una clase general y luego extenderla a clases más especializadas. Las clases especializadas heredan las propiedades y métodos de la clase general. Considera objetos geométricos. Supongamos que desea diseñar las clases para modelar objetos geométricas como círculos y rectángulos. Los objetos geométricos tienen muchas propiedades comunes y comportamientos. Se pueden dibujar en un determinado color, con o sin relleno. Por lo tanto, una clase general GeometricObject se puede usar para modelar todo objeto geométrico.
